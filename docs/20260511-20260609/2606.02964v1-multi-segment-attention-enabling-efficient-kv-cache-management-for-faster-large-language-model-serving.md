## 摘要
大型语言模型（LLM）推理依赖键值（KV）缓存来避免冗余的注意力计算。虽然近似的KV缓存保留技术通过牺牲模型精度来减少内存使用，但无损方法则将KV缓存块从GPU内存中驱逐，并按需重建以保留精确输出。现有的无损KV缓存管理系统主要基于访问频率或位置启发式方法做出驱逐决策，而未考虑不同KV缓存块如何影响GPU注意力内核的执行效率。
本文提出AsymCache，一种面向LLM推理的计算延迟感知KV缓存管理系统，其明确将缓存驻留决策与GPU注意力内核性能对齐，包含三个关键组件：用于高效非连续KV上下文处理的多段注意力（MSA）、联合优化命中率和位置感知重计算成本的缓存驱逐策略，以及用于高硬件利用率的自适应分块调度器。实验表明，与最新基线相比，AsymCache将首次令牌时间（TTFT）降低高达1.90-2.03倍，每输出令牌时间（TPOT）降低1.62-1.71倍，证实了该方法在常见工作负载中的有效性，并验证了其平衡计算效率与缓存命中率的设计目标。此外，AsymCache的低级设计允许无缝集成到如Continuum这样的智能体服务系统中，进一步将平均任务延迟降低高达18.1%。

## Abstract
Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.   In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

## 2. 论文提出的方法论

### 核心思想
- **AsymCache**：一个面向LLM推理的计算延迟感知KV缓存管理系统，其核心是**优先保留重计算代价高且未来复用概率大的缓存块**，而驱逐代价低且复用概率低的块，从而在整体预期延迟上取得最优。

### 关键技术细节
#### (1) Multi-Segment Attention (MSA) — 多段注意力内核
- 支持非连续KV上下文（例如同时缓存前缀和后缀，中间块被驱逐），允许一个请求在单个CUDA内核中同时处理多个不连续的已缓存段和待计算段。
- 实现方式：将不同请求分配到不同CTA（合作线程数组）组，通过预计算数组获取每个token块的实际序列长度，避免多次内核启动开销。

#### (2) Computational-Aware Block Evictor — 计算感知块驱逐器
- 驱逐准则：选择**期望重计算延迟最小**的块进行驱逐，即最小化 \( f_B(t) \cdot \Delta T_B \)，其中：
  - \( f_B(t) \)：历史访问频率（带指数衰减），反映未来复用概率。
  - \( \Delta T_B \)：该块被驱逐后需重计算带来的额外延迟，由**位置**决定（后缀块代价更大）。
- 采用**分段指数函数**作为频率项，以满足**顺序保持规则**（order-preserving rule），从而支持使用平衡树实现 \( O(\log n) \) 的驱逐复杂度，优于线性复杂度方法。
- 引入在线可调系数 \( \lambda \)，自适应调整频率函数的有效转折点，适应不同工作负载的寿命分布变化。

#### (3) Chunking Scheduler — 自适应分块调度器
- 基于chunked-prefill思想，将长prefill请求拆分为小chunks，避免头阻塞。
- 允许chunk跨越多个不连续段（包括已缓存段和待计算段），与MSA内核协同工作。
- 在高负载时动态减小chunk大小以降低TPOT，同时设定下界避免GPU欠利用。

### 整体算法流程（文字说明）
1. 用户请求到达，系统检查其与历史对话的缓存匹配情况，可能触发驱逐。
2. 驱逐器根据每个缓存块的 \( f_B(t) \cdot \Delta T_B \) 值，利用两个平衡树分别维护分段指数函数的两段，选择最小值块驱逐。
3. 请求被批处理，chunking scheduler为每个prefill请求分配chunk大小，chunk可能跨越已缓存段和待计算段。
4. 批处理提交给MSA内核，在单个内核中并行计算多个非连续段注意力。
5. 输出结果，并更新缓存块的访问频率。

## 6. 论文主要结论与发现

1. **AsymCache显著优于现有基线**：在低/高分散工作负载下，TTFT降低1.90–2.03倍，TPOT降低1.62–1.71倍（相对于最新基线）。

3. **多段注意力内核消除内核启动开销**：单内核融合处理非连续上下文，使后缀缓存策略在短序列下也具备竞争力。
4. **对数时间驱逐算法必要且高效**：线性时间算法会导致可观的控制平面开销（~200ms/请求），对数算法降至~2ms，且性能进一步提升。

6. **超参数鲁棒**：在合理范围内（寿命、复用概率、变化率）性能稳定，仅极端值有轻微退化。
