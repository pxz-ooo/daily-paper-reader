## 摘要
推测性解码（SD）通过采用草稿-验证范式加速低并发LLM推理。然而，主流方法通常依赖多令牌预测，这引入了递增的预测难度和串行草稿延迟。为了解决这些问题，我们提出了推测性流水线解码（SPD），这是一个开创性的框架，释放了流水线并行的真正潜力。通过将目标LLM划分为n个流水线阶段，SPD允许LLM并行处理n个令牌以加速解码。为了在单序列解码中持续填充流水线，一个推测模块聚合跨不同流水线深度的中间特征以预测下一个令牌，与目标模型的流水线步骤严格并行执行，从而实现有界难度、更高的接受率和零延迟气泡。我们的实验表明，与主流基线相比，SPD实现了显著更高的理论加速比，为LLM解码加速提供了一种高度可扩展的解决方案。我们的代码可在https://github.com/yuyijiong/speculative_pipeline_decoding获取。

## Abstract
Speculative Decoding (SD) accelerates low-concurrency LLM inference by employing a draft-then-verify paradigm. However, mainstream methods typically rely on multi-token prediction, which introduces escalating prediction difficulty and serial drafting latency. To address these, we propose Speculative Pipeline Decoding (SPD), a groundbreaking framework that unlocks the true potential of pipeline parallelism. By partitioning the target LLM into $n$ pipeline stages, SPD allows LLM to process $n$ tokens in parallel to accelerate decoding. To continuous fill the pipeline in single sequence decoding, a speculation module aggregates intermediate features across different pipeline depths to predict the next token, executing strictly in parallel with the target model's pipeline step, to realize bounded difficulty, higher acceptance rates, and zero latency bubbles. Our experiments demonstrate that SPD achieves a significantly higher theoretical speedup compared to mainstream baselines, offering a highly scalable solution for LLM decoding acceleration. Our code is available at https://github.com/yuyijiong/speculative_pipeline_decoding

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：主流推测性解码（Speculative Decoding, SD）方法，如EAGLE系列，依赖“多token预测”范式。该范式存在两个根本性的结构性缺陷：
    1.  **预测难度递增**：当草稿模型预测越远的未来token时，由于依赖自身浅层且未经验证的隐藏状态，其与目标模型真实分布的偏差会迅速累积（长程衰减/分布外问题），导致后续草稿token的接受率急剧下降，深层推测效率低下。
    2.  **串行延迟与相互等待**：草稿token的序列化生成迫使目标模型在草稿阶段空闲，造成计算气泡，侵蚀了加速比。
- **研究背景**：为缓解这些问题，学界出现了并行草稿生成（如P-EAGLE）、异步推测（如Speculative Speculative Decoding）以及基于流水线并行的自推测解码（如PPSD）等方向。但这些方法要么引入巨大的训练开销和精度风险，要么因草稿模块复杂度暴增而带来计算和显存瓶颈，或者（如PPSD）因仅依赖浅层特征而接受率低且无法消除串行等待。
- **整体含义**：本论文旨在从根本上解决上述瓶颈，通过引入一种全新的**流水线并行范式**，打破传统多token预测的框架，实现**预测难度有界、草稿延迟零开销、加速比高度可扩展**的LLM推理加速方案。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**Speculative Pipeline Decoding (SPD)**。将目标LLM划分为n个流水线阶段，使其能够并行处理n个token。核心在于利用一个轻量级的**推测模块（Speculation Module）**，在目标模型进行流水线步骤的同时，并行地、单步地预测下一个token，从而填充流水线，实现零气泡推断。
- **关键技术细节**：
    1.  **流水线执行框架**：将目标LLM的L层均匀划分为n个阶段。每个解码周期，所有活跃token都向前推进一个阶段，同时一个完全处理完毕的token退出流水线，一个新token进入。
    2.  **多深度特征聚合 (Multi-Depth Feature Aggregation, 公式1)**：这是解决预测难度递增的关键。不同于PPSD仅利用第一阶段浅层特征，SPD的推测模块会根据每个token当前所处的流水线深度（k），聚合其从嵌入层到最深已完成层的三个均匀间隔层的隐藏状态（嵌入层、中间层、最深计算层）。该特征表示为：
        $$g^k_t = \text{FC}\left( \text{Concat}\left[ H^0_t, H^{l_{max}/2}_t, H^{l_{max}}_t \right] \right)$$
        这使得推测过程总能获得尽可能丰富、上下文对齐的深层信息，将预测误差上界严格限定在固定的流水线深度n内。
    3.  **零等待同步执行调度 (Simultaneous Execution Schedule, 公式2)**：这是实现零气泡的关键。推测模块的输入不依赖流水线步骤的“输出状态”，而是在新token进入流水线前的“输入状态”就开始推测。其输入序列特征Gt为：
        $$G_t = [g^n_1, g^n_2, \dots, g^n_{t-n}, g^{n-1}_{t-n+1}, \dots, g^1_{t-1}, g^0_t]$$
        这种方式下，推测模块的计算与目标模型下一个流水线步骤完全并行。只要推测模块的延迟不超过单个流水线阶段的执行时间，其计算成本就被完美隐藏。
    4.  **验证和回滚机制**：验证以流式方式进行。当最老token退出流水线时，使用其完全计算出的logits验证n个周期前就已在流水线中的token。若接收，则继续；若拒绝，则触发一个回滚协议，清除KV Cache和流水线中间状态，并用正确的token重新填充。
- **算法流程（文字描述）**：在每一步，推测模块基于当前所有token所处的阶段特征，预测下一个token。该预测值与目标模型流水线步骤并行进行。流水线步骤完成后，新退出token的logits用于验证流水线中最老但尚未出队token的正确性。若通过，新预测的token进入流水线；若失败，则执行回滚。

- **SPD实现更高的理论加速比**：在大部分测试配置下（尤其是在高温度采样T=1和单路径推测W=1时），SPD的理论加速比显著优于EAGLE-3和PPSD。尽管EAGLE-3的原始接受长度更高，但SPD通过零隐藏延迟的优势，赢得了最终加速比。

- **缺乏端到端壁钟时间测量**：当前实现基于原生PyTorch，未进行系统级优化（如CUDA核心、稀疏注意力、异步执行），无法提供生产级的性能数据。理论加速比与实际加速比可能存在差距，尤其是当多个流水线阶段运行在单GPU上时，会遇到内存带宽瓶颈和内核启动开销。
