## 摘要
具有扩展上下文长度的大语言模型依赖键值（KV）缓存来支持对先前标记的注意力。然而，维护KV缓存会带来显著的内存开销，从而促使通过驱逐和合并来强制执行固定预算的KV缓存压缩方法。现代驱逐方法越来越多地采用基于跨度的保留，因为保留连续跨度在实践中有效且更好地保持语义连贯性。然而，当与驱逐后合并结合时，基于跨度的保留将合并集中到一小部分跨边界载体标记上，产生高度不平衡的合并模式，加剧了过度合并并增加了信息损失。为了解决这种不平衡，我们提出了GRKV（KV缓存的全局回归），一种免训练的KV缓存合并方法，直接最小化压缩缓存与完整缓存注意力输出之间的差异。GRKV使用基于岭回归的合并步骤，将来自被驱逐标记的信息分布到保留标记上，同时正则化更新以防止过度平滑。在LongBench和RULER长上下文基准测试中，GRKV是唯一以最小开销提升整体性能的合并方法。

## Abstract
Large language models (LLMs) with extended context lengths rely on the key-value (KV) cache to support attention over prior tokens. However, maintaining the KV cache incurs substantial memory overhead, motivating KV-cache compression methods that enforce a fixed budget through eviction and merging. Modern eviction methods increasingly adopt span-based retention because preserving contiguous spans is empirically effective and better preserves semantic coherence. Yet, when combined with post-eviction merging, span-based retention concentrates merges onto a small set of span-boundary carrier tokens, producing a highly imbalanced merge pattern that exacerbates over-merging and increases information loss. To address this imbalance, we propose GRKV (Global Regression for KV Cache), a training-free KV-cache merging method that directly minimizes the discrepancy between compressed-cache and full-cache attention outputs. GRKV uses ridge-regression-based merge steps to distribute information from evicted tokens across retained tokens, while regularizing the updates to prevent over-smoothing. Across the LongBench and RULER long-context benchmarks, GRKV is the only merging method that improves overall performance with minimal overhead.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型处理长上下文时，需要维护键值（KV）缓存来支持对先前令牌的注意力操作，但缓存占用大量内存，从而催生了KV缓存压缩技术。
- **现有方法**：主要分为驱逐（eviction）和合并（merging）。现代驱逐方法倾向于采用基于跨度（span-based）的保留策略（如SnapKV），认为保留连续文本段能更好地维持语义连贯性。
- **问题**：当将跨度过保与后续合并结合时，合并操作过度集中在少量跨度边界令牌上，形成高度不平衡的合并模式。这导致边界令牌承载过多被驱逐令牌的信息，造成“过度合并”（over-merging），增加信息损失，降低模型性能。
- **核心意义**：提出一种全局、平衡、免训练的KV缓存合并方法，直接优化压缩缓存与完整缓存的注意力输出一致性，从而缓解不平衡合并带来的性能下降。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：将KV缓存合并形式化为一个全局回归问题，直接最小化压缩缓存与完整缓存之间注意力输出的差异（代替局部启发式规则），并通过岭回归（ridge regression）正则化防止更新后的缓存过度偏离初始保留缓存。
- **关键技术细节**：
  - **目标函数**：对每个查询 \(i\)，定义偏差损失 \(L_i = \| \mathbf{o}_i^{(\text{full})} - \mathbf{o}_i^{(\text{Ret})} \|_2^2\)，并简化为 \(\|\Delta_i\|_2^2\)（省略输出投影矩阵），其中 \(\Delta_i = \mathbf{A}_i^{(\text{full})}\mathbf{V}_{\text{full}} - \mathbf{A}_i^{(\text{Ret})}\mathbf{V}_{\text{Ret}}\)。
  - **代理查询窗口**：由于未来查询未知，使用提示（prompt）末尾的短查询窗口（大小 \(m=32\)）作为代理。论文通过实验证明不同窗口的注意力输出高度一致，因此该窗口能稳定代表未来查询。
  - **岭正则化**：在目标中加入 \(\lambda_k \|\mathbf{K}_{\text{Ret}} - \mathbf{K}_{\text{Ret}}^0\|_F^2\) 和 \(\lambda_v \|\mathbf{V}_{\text{Ret}} - \mathbf{V}_{\text{Ret}}^0\|_F^2\)，防止过度优化导致缓存漂移。
  - **交替优化**：
    - **值（Value）步骤**：固定键，值更新为闭式岭回归解，利用Woodbury恒等式将矩阵求逆从 \(c \times c\) 降为 \(m \times m\)（\(c\) 为缓存预算，远大于 \(m\)）。
    - **键（Key）步骤**：固定值，将包含softmax的非线性函数局部线性化（一阶泰勒展开），然后求解岭回归，同样用对偶形式降低维度。
  - **固定令牌**：优化过程中保留sink令牌、代理窗口内查询令牌以及注意力分数最高的 \(\beta=10\%\) 的保留令牌不变，以保持生成稳定性。
- **算法流程**（文字描述）：
  1. 输入：完整缓存 \(\mathbf{K}_{\text{full}},\mathbf{V}_{\text{full}}\)，初始保留缓存 \(\mathbf{K}_{\text{Ret}}^0,\mathbf{V}_{\text{Ret}}^0\)，代理查询窗口 \(\mathbf{Q}_{\text{win}}\)。
  2. 固定部分令牌集 \(\mathcal{F}\)，其余为自由令牌 \(\mathcal{G}\)。
  3. 计算完整缓存目标 \(\mathbf{Y} = \text{softmax}(\mathbf{Q}_{\text{win}}\mathbf{K}_{\text{full}}^\top/\sqrt{d})\mathbf{V}_{\text{full}}\)。
  4. 迭代 \(S\) 次（默认为1）：
     - 值步骤：用自由令牌对应的子矩阵求解闭式解更新 \(\mathbf{V}_{\text{Ret}}\)，固定令牌保持不变。
     - 键步骤：线性化注意力输出，求解对偶系统更新 \(\mathbf{K}_{\text{Ret}}\)，固定令牌保持。
     - 检查收敛（\(\Delta \leq 10^{-9}\)）则提前终止。
  5. 返回最终合并后的 \(\mathbf{K}_{\text{Ret}}^*,\mathbf{V}_{\text{Ret}}^*\)。

## 3. 实验设计：数据集、基准与对比方法

- **数据集**：

  - **RULER**：包含13个任务（16K上下文长度），评估提取、检索、问答和变量追踪。
- **基准**：
  - 全缓存（Full Cache）作为理想参考。
  - 驱逐方法（span-based）：SnapKV、CriticalKV、PyramidKV、Ada-KV。
  - 驱逐方法（token-based）：H2O。
  - 合并方法（baseline）：CaM、D2O、AsymKV。
- **模型**：
  - Llama-3.1-8B-Instruct（最高128K）、Mistral-7B-Instruct-v0.3（最高32K）、Qwen3-14B（最高128K）。

  - 主要使用10%缓存预算，另在20%预算下测试。
  - 合并方法均基于基础驱逐方法（如SnapKV+GRKV）。

## 6. 主要结论与发现

- **GRKV是唯一在所有设置下提升整体平均分数的合并方法**：在LongBench和RULER上，GRKV始终优于基础驱逐方法（SnapKV, CriticalKV），而CaM、D2O、AsymKV往往导致性能下降。
- **公开账期正则化至关重要**：去掉正则化会导致分数明显下降（LongBench 34.58→32.06）。
- **单步交替更新最优**：增加迭代步数不会提升性能，可能过拟合代理窗口。
- **固定少量高注意力令牌**（\(\beta=10\%\)）和sink/window令牌能稳定优化。
- **同时更新键和值效果最好**：单独更新键或值均有提升，但联合更新最佳。
- **额外开销极小**：主要增加在预填充阶段，解码阶段几乎无额外延迟，内存占用与基础驱逐方法相近。

## 7. 优点

- **方法论亮点**：
  - 将KV合并建模为全局回归问题，直接优化与完整缓存的输出一致性，而非依赖局部启发式规则。
