## 摘要
低秩投影通过利用隐藏维度冗余已成为压缩KV缓存的一种有前途的方法。然而，先前的方法依赖于固定或启发式秩选择，并且难以在最小化精度下降的同时实现激进压缩。我们提出了STAR-KV，一种具有细粒度秩控制的自适应低秩KV缓存压缩框架。STAR-KV包括：1) 一种可微的阈值机制，能够在注意力头和块级别实现最优秩选择；2) 一种混合分解策略，根据键和值投影的敏感性应用不同的低秩分解；3) 一种低秩感知的混合精度量化，利用数据统计实现近乎无损的低比特量化。在多个LLM和基准测试中评估，STAR-KV实现了高达75%的KV缓存压缩，以及与量化结合时高达20倍的整体KV缓存减少。通过基于自定义Triton的GPU内核，STAR-KV在注意力模块上实现了高达6.9倍的加速，端到端生成吞吐量提升3.1倍。我们的代码公开于：https://github.com/PriyanshBhatnagar/STAR-KV。

## Abstract
Low-rank projection has emerged as a promising approach for compressing the KV cache by exploiting hidden-dimension redundancy. However, prior methods rely on fixed or heuristic rank selection and struggle to achieve aggressive compression with minimal accuracy degradation. We propose STAR-KV, an adaptive low-rank KV cache compression framework with fine-grained rank control. STAR-KV encompasses 1) a differentiable thresholding mechanism that enables optimal rank selection at both attention-head and block levels, 2) a hybrid decomposition strategy that applies different low-rank factorizations according to the sensitivity of key and value projections, and 3) a low-rank-aware mixed precision quantization that leverages data statistics for near lossless low-bit quantization. Evaluated across multiple LLMs and benchmarks, STAR-KV achieves up to 75% KV cache compression and up to 20x overall KV cache reduction when combined with quantization. Enabled by custom Triton-based GPU kernels, STAR-KV delivers up to 6.9x speedup for the attention module and 3.1x end-to-end generation throughput. Our code is publicly available at: https://github.com/PriyanshBhatnagar/STAR-KV.

## 1. 核心问题与整体含义（研究动机和背景）

大语言模型推理时，KV缓存的内存占用随序列长度线性增长，成为长上下文推理的主要瓶颈。低秩投影通过利用隐藏维度的冗余来压缩KV缓存，但现有方法（如 Palu、ReCalKV）采用固定或启发式秩选择，导致高压缩率下精度显著下降。STAR-KV 旨在解决“如何在激进压缩下保持模型精度”的核心问题，通过引入自适应秩控制框架，实现更优的精度-压缩权衡。

## 2. 方法论：核心思想与关键技术细节

### 核心思想
- 将秩选择参数化为**可微软阈值（soft thresholding）**，作用于奇异值谱，使得每个注意力头和每个解码器块能自动学习最优秩。
- 结合**混合分解策略**：对键投影采用头级分解（HD，低重建开销），对值投影采用联合分解（JD，低近似误差）。
- 利用低秩表示固有的有序奇异值特性，设计**低秩感知的混合精度量化**，通过块级Hadamard变换和离群/内敛分块实现近无损低比特量化。

### 关键技术细节
- **软阈值算子**：  
  `Ths(x;α) = x·tanh(s(x-α))`（x>=α）或 `c·tanh(s(x-α))`（x<α，c=0），平滑近似硬截断，可微，允许梯度优化。
- **训练目标**：  
  `L_total = L_KD + γ·L_acmp`  
  - L_KD：知识蒸馏KL散度，教师为未压缩模型，学生为压缩模型。  
  - L_acmp = Σ e^{-α}：自适应压缩损失，α增大时梯度衰减，使模型优先稳定精度。
- **混合分解**：  
  - 键：每头独立分解（HD），重建FLOPs为 `l·c·h·dh²`。  
  - 值：所有头联合分解（JD），重建FLOPs为 `l·c·h²·dh²`。  
  值采用JD可获得更低近似误差（由Eckart-Young-Mirsky引理证明），且通过**权重吸收（reordering）** 将值重建矩阵U_v与输出投影W_out合并，大幅降低计算开销。
- **低秩感知量化**：  
  将低秩表示的通道按奇异值大小分为离群块（前20%）和内敛块（后80%），分别用b_out=4bit和b_in=3bit量化（平均3.2bit）。对每个块独立应用Hadamard变换，平滑异常值。Hadamard矩阵可融合入投影矩阵，无额外运行时开销。

- **精度-压缩权衡最优**：在LongChat-7B上，75%低秩压缩仅损失1.04%零样本平均准确率（60.49% vs 基线61.53%），远优于Palu（50%压缩已降至58.49%）。

- **系统加速**：注意力模块最高6.9x加速（64K上下文，估计基线），端到端生成吞吐量提升3.1x（平均）。

## 7. 优点（方法/实验亮点）

- **自适应秩控制**：可微阈值首次实现数据驱动的逐头/逐块秩选择，优于静态和启发式方法。
- **混合分解**：基于理论分析（重建开销、近似误差）和实验验证，找到精度-速度最优组合（HD-K + JD-V）。
- **低秩感知量化**：利用奇异值有序性自然区分通道重要性，避免复杂异常值建模。
- **系统优化**：自定义Triton内核融合解量化、RoPE、重建和注意力计算，并采用GEMM替代GEMV，提升硬件利用率。
- **迁移性**：学习到的秩配置在校准数据集和相近模型间稳定，降低了多次校准的成本。

## 8. 不足与局限

- **训练开销**：虽仅需<6 GPU小时，但仍依赖校准数据微调，与完全免训练方法（如静态SVD）相比增加了部署流程。
- **模型覆盖**：主要测试7B~13B模型，未涉及更大规模（如70B）或MoE架构；GQA模型（如LLaMA-3.1）在极长上下文（16K RULER）仍有下降，需进一步研究。
- **量化精度**：3.2bit量化引入额外误差，在部分任务（如HellaSwag）上下降略大；未与更精细的非均匀量化（如KVQuant的2bit）直接对比（因为后者精度已更低）。
- **静态秩选择假设**：软阈值在训练后固定为硬截断，无法在线动态适应序列上下文；而近期研究（如PyramidKV）支持分层动态压缩。
- **硬件依赖性**：加速效果高度依赖Triton内核和NVIDIA GPU，未在AMD/其他加速器上验证。
