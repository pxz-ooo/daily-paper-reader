## 摘要
大型语言模型中使用的KV缓存具有线性增长的时间复杂度，因此在处理长上下文时，LLMs面临内存爆炸和解码效率降低的问题。当前的KV缓存驱逐已成为重要的研究方向；然而，基于固定软令牌（如Judge Q）的现有方法依赖静态参数集作为查询来评估KV对的重要性，因此无法动态适应不同的输入提示，也无法精确捕捉复杂且不断变化的任务相关性。此外，被驱逐的KV对被永久丢弃，导致不可逆的信息丢失和上下文断裂。为解决这一问题，我们提出了Meta-Soft，一种基于探针驱动上下文集成的动态压缩框架。具体来说，我们构建了一个带有可学习正交基矩阵L的元库，并使用带有Gumbel-Softmax的选择器网络生成可微分的稀疏组合权重，从而从输入提示特征中动态合成最具针对性的k个软令牌。我们将这些软令牌附加到输入序列的末尾以探询关键信息。我们还引入了一种基于注意力流的集成机制，将移除令牌的语义信息重新分配到保留令牌中，从而有效保留丢弃的上下文信息。在多个数据集上的实验表明，我们的方法优于现有的最先进驱逐方法，并为KV缓存压缩提供了新的解决方案。

## Abstract
The KV cache used in large language models has linearly growing time complexity, so LLMs face memory blow-up and reduced decoding efficiency when they process long contexts. Current KV Cache eviction has become an important research direction; however, existing methods based on fixed Soft Tokens (e.g., Judge Q) rely on a static parameter set as the query to evaluate the importance of KV pairs, so they cannot adapt dynamically to different input prompts, and they cannot precisely capture complex and changing task relevance. Also, evicted KV pairs are discarded permanently, so this causes irreversible information loss and context breaks. To address this problem, we propose Meta-Soft, a dynamic compression framework based on probe-driven context integration. Specifically, we build a meta-library with a learnable orthogonal basis matrix $\mathcal{L}$, and we use a selector network with Gumbel-Softmax to produce differentiable sparse combination weights, so we dynamically synthesize the most targeted $k$ Soft Tokens from the input prompt features. We append these Soft Tokens to the end of the input sequence to probe key information. We also introduce an attention-flow based integration mechanism, which redistributes the semantic information of removed tokens into retained tokens, and this keeps the dropped context information effectively. Experiments on multiple datasets show that our method outperforms existing state-of-the-art eviction methods and provides a new solution for KV Cache compression.

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：通过离线训练一个可学习的正交基元库（Meta-Library）和轻量级选择器网络，在线推理时根据输入提示动态合成特定任务的最优软令牌（Soft Tokens），作为探针查询全局 KV 重要性；再通过注意力流将淘汰令牌的语义信息整合到保留令牌中，实现无损压缩。
- **关键技术细节**：
  - **元库与选择器**：
    - 元库：一个可学习的正交基矩阵 $\mathcal{L} \in \mathbb{R}^{M \times d}$（$M=512$），保证特征多样性（通过正交正则化）。
    - 选择器网络 $f_\theta$：利用 Gumbel-Softmax 生成稀疏组合权重，从元库中动态选取 $k=32$ 个基向量并加权求和，合成输入相关的软令牌。
  - **探针与缓存分区**：
    - 将合成软令牌 $E_{soft}$ 拼接到输入嵌入末尾，通过一次前向注意力计算得到探针得分 $S = Q_{probe} (K_{prompt})^T / \sqrt{d_k}$，经 Softmax 和平均得到全局重要性分布 $A_{soft}$。
    - 根据 $A_{soft}$ 选择 Top-B 保留（$I_{keep}$），其余为淘汰集（$I_{drop}$）。
  - **上下文整合（Contextual Consolidation）**：
    - 利用注意力流计算淘汰令牌与保留令牌的键空间相似度 $S_{sim} = K_{drop} K_{keep}^T / \sqrt{d_k}$。
    - 采用稀疏负载均衡路由：对每个淘汰令牌只关联 Top-m 个最相似保留令牌，并引入列重加权惩罚过载保留令牌，防止信息塌缩。
    - 最终将淘汰令牌的值按加权和 $\Delta V = W_{flow}^T V_{drop}$ 累加到保留令牌的 $V$ 上，保持 $K$ 不变。

## 6. 主要结论与发现
- Meta-Soft 在 PG19 和 OpenWebText2 上困惑度低于所有基线（如 16K 时 PPL 7.49 vs. Judge Q 的 7.58），证明更好的语言保真度。
- 在 LongBench 上，Meta-Soft 在两个 backbone 上均达到平均最高分（B=256 时 47.19 vs. AnDPro 46.98），保留 Full KV 的 92.9%–97.2% 性能。
- 在 RULER 上，Meta-Soft 在所有上下文长度 4K–128K 上平均得分 75.72，优于 AnDPro 的 74.85，且随长度增长退化更温和。
- 消融实验显示：动态软令牌（DST）和注意力流聚合（AFA）均有贡献，二者结合效果最佳（RULER +11.24，LongBench +1.51 vs. 基线）。
- 效率方面：软令牌生成仅增加 0.32–2.34 ms（<0.3% 预填充时间），端到端延迟与基线相当，支持更大批大小和更高吞吐（80 batch vs. Full KV 的 28 batch，吞吐 390.17 token/s vs. 180.49 token/s）。

## 7. 优点
- **动态适应性**：基于输入提示动态合成软令牌，克服了静态查询方法的跨任务失配问题。
- **信息保持**：注意力流整合替代硬淘汰，有效保留上下文语义，避免断裂和幻觉。
- **即插即用**：不需要修改 LLM 骨干，仅需预填充阶段额外计算，解码阶段零额外开销。
- **可扩展性**：元库+选择器机制支持无监督学习（基于 ground-truth 注意力），且正交正则化保证表示多样性。
- **综合表现**：在多个基准上达到 SOTA，特别是在长距离检索和内存受限场景下优势明显。

## 8. 不足与局限
- **实验覆盖**：未在更大型模型（如 70B）或更多 backbone（如 GPT 类）上验证；未评估在更长上下文（如 256K+）或多轮对话场景下的表现。
- **偏差风险**：训练数据来自 SlimPajama 和 ShareGPT，可能引入数据分布偏差，影响泛化性。

- **评估指标**：主要依赖困惑度和下游任务得分，未进行人工评估或分析生成文本质量（如事实一致性）。
