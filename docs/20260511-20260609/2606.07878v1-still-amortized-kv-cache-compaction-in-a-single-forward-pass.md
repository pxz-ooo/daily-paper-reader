## 摘要
KV缓存是长时程语言模型部署的内存瓶颈。实际上，可部署的压缩器必须足够轻量以便在推理时调用，足够有表现力以在约束下保留上下文，并且能够在轨迹中重复使用。现有的压缩方法仅满足部分要求：选择方法轻量但受限于子集，而合成方法有表现力但依赖于逐上下文优化。本文提出Still，一种针对冻结基模型一次性训练的小型逐层Perceiver，能够在单次前向传播中生成紧凑的键和值。在Qwen和Gemma模型上，Still在压缩比从8倍到200倍、上下文长度从8k到128k的范围内占据速度-质量前沿的有利位置。在长上下文RULER网格中，Still超过最强基线8-22分。相同的紧凑缓存还支持自由形式的摘要，在HELMET上保留了大部分全上下文增益，并在与KV-Distill的成对LongBench摘要比较中胜出。由于压缩是一次前向传播，Still可以迭代应用，进入逐上下文方法无法实现的长期运行状态。我们表明，摊销使得长上下文缓存压缩变得可行，而合成使其压缩状态在极端压缩下依然有用。

## Abstract
The KV cache is the memory bottleneck of long-horizon language model deployment. Practically, a deployable compactor must be lightweight enough to call during inference, expressive enough to preserve context under constraint, and reusable across a trajectory. Existing compaction methods satisfy only part of this requirement: selection methods are lightweight but subset-bound, while synthesis methods are expressive but rely on per-context optimization. Here we introduce Still, a small per-layer Perceiver trained once against a frozen base model that produces compact keys and values in a single forward pass. On Qwen and Gemma models, Still occupies the favorable side of the speed--quality frontier across compression ratios from $8\times$ to $200\times$ and context lengths from $8$k to $128$k. On the long-context RULER grid, Still exceeds the strongest baseline by 8--22 points. The same compact cache also supports free-form summarization, preserving most of the full-context gain on HELMET and winning a pairwise LongBench summarization comparison against KV-Distill. Because compaction is a forward pass, Still can be applied iteratively, entering a long-horizon regime unavailable to per-context methods. We show that amortization makes long-context cache compaction tractable, and synthesis makes its compact state useful at extreme compression.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：长时程语言模型部署中，KV缓存是内存瓶颈。现有的KV缓存压缩方法要么是“选择型”（轻量但只能保留原始token子集，受限于子集表示能力），要么是“合成型”（表达力强但需要为每个上下文单独优化，不适合重复使用）。
- **背景**：理想的可部署压缩器应同时满足：推理时轻量、有表现力、可跨轨迹重用。现有方法无法三者兼顾。
- **整体含义**：Still通过将KV缓存压缩抽象为学到的变换，在**单次前向传播**中完成压缩（摊销化），避免了逐上下文优化，同时通过合成新键值对突破了子集选择的天花板，使得长上下文压缩变得实用。

## 2. 方法论

### 核心思想
- 针对**冻结的预训练基模型**，训练一个小型**逐层Perceiver压缩器**。
- 压缩器接受全量KV缓存（去旋转后的键），输出紧凑的键`Ck`和值`Cv`（`t << T`），在单次前向传播内完成。
- 压缩后的缓存替换原始前缀缓存，用于后续token的解码。

### 关键技术细节
- **Per-layer Perceiver架构**：每层一个独立的压缩器模块。输入为每头的键值拼接（先去RoPE），使用一组可学习的**潜变量查询**（per-head）进行交叉注意力、自注意力和FFN（可选），最后通过共享的线性投影头输出紧凑键值。
- **位置无关压缩**：先将原始键逆旋转为无位置帧，压缩器内部使用自己的RoPE，并在输出时重新旋转到选定位置，避免内容与相位混合。
- **单次压缩与迭代压缩**：
  - 单次：对静态前缀一次预填充得到全量缓存，应用压缩，后续生成使用压缩缓存。
  - 迭代：上下文分块到达，每块预填充后压缩前一块，保留一个原始块作为“lookahead”缓冲，实现固定压缩比的渐进式缓存增长。
- **训练**：训练信号为前向KL散度（只对答案侧tokens），从全量缓存教师蒸馏到压缩缓存学生。教师和学生是同一冻结模型，仅缓存不同。损失仅在答案token上计算（含推理轨迹），并截取top-200词汇以控制内存。

### 公式/算法（文字说明）
- 核心损失：`L = E_{(c,p,a)~D} Σ_{i∈ans} KL( f_θ(·|c,p,a_{<i}) ∥ f_θ(·|g_φ(c),p,a_{<i}) )`
- 压缩器输出：`(C_k, C_v) = g_φ(K, V)`，其中`C_k, C_v ∈ R^{t×d}`，`t << T`。

## 6. 主要结论与发现
- **速度-质量前沿**：Still在所有压缩比和上下文长度下处于有利侧，是唯一同时保持高精度和快速的方法。选择型方法在高压缩比下快速退化。
- **迁移能力**：同一压缩器可直接迁移到不同规模Qwen3（4B-32B）和MoE架构，以及对Gemma-3仅压缩全局注意力层后超越定长缓存基线。
- **长上下文RULER**：在匹配训练条件下，Still超越KV-Distill 8-22分（16/18组）；在零样本迁移下也保持领先。

- **迭代压缩**：可以按固定压缩比逐块迭代，但训练长度限制了可扩展范围（8k训练在128k下崩溃，32k训练保持可用）。
- **摊销合成**：摊销使压缩可调用多次，合成使紧凑状态在极端压缩下仍有表达力。

## 7. 优点
- **单次前向传播**：无需每上下文优化，推理时额外开销低。
- **合成表达力**：突破子集选择限制，可混合任意多位置信息。
- **无模型修改**：冻结基模型，仅添加小模块，易于部署。
- **多场景统一**：单一方法同时支持问答、摘要、迭代长文本。
- **充分消融**：对架构、损失函数、β偏置、键值分解、词汇支持等进行了详细消融。
- **公平对比**：与最强摊销选择方法KV-Distill在同一数据、同一模型、同一训练协议下比较，报告多种子标准差。

## 8. 不足与局限
- **非无损**：在128k最难设置下，全上下文仍显著更好；迭代压缩不是自由外推，训练长度决定了可承受的递归次数。
