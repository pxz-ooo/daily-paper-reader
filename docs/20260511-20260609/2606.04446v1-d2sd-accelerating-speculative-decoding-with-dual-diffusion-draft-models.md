## 摘要
推测解码通过草拟多个令牌并在一次目标模型前向传递中验证它们，从而加速自回归大语言模型推理。最近的基于扩散的草稿器并行生成整个令牌块，但通常每次验证只提交一个草稿序列：一旦出现第一个不匹配，所有后续草稿令牌都被丢弃，导致接受率有限。简单地对更多草稿候选序列进行批处理仅带来边际改进，因为冗余或位置不当的分支增加了草拟和验证的成本，而接受的令牌数量并未成比例增加。我们提出D^2SD，一种双扩散草稿推测解码框架，该框架将候选组织成基于置信度的前缀树，其中第一个扩散草稿器生成一个块及每个位置的置信度分数，用于识别最可能的拒绝边界，并选择前K个前缀范围进行恢复；第二个可变前缀扩散草稿器在每个选定的前缀处重新锚定，并在一次批处理传递中提出替代续接；生成的共享前缀候选通过级联注意力联合验证。实证结果显示，D^2SD相对于底层扩散方法和强自回归推测解码基线均有明显改进。

## Abstract
Speculative decoding accelerates autoregressive large language model inference by drafting multiple tokens and verifying them in a single target-model forward pass. Recent diffusion-based drafters generate an entire block of tokens in parallel but usually commit to a single draft sequence per verification: once the first mismatch occurs, all subsequent draft tokens are discarded, resulting in a limited acceptance rate. Naively batching more draft candidate sequences only introduces a marginal improvement, as redundant or poorly placed branches increase the cost of drafting and verification without proportionally increasing the number of accepted tokens.   We propose D^2SD, a dual diffusion draft speculative decoding framework that organizes candidates into a confidence-guided prefix tree, where the first diffusion drafter generates a block along with per-position confidence scores that are used to identify the most likely rejection boundary and select the top-K prefix ranges for recovery; the second variable-prefix diffusion drafter re-anchors at each selected prefix and proposes alternative continuations in one batched pass; the resulting shared-prefix candidates are jointly verified via cascade attention. Empirically, D^2SD shows clear improvements over both the underlying diffusion approach and strong autoregressive speculative decoding baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在推测解码中，现有扩散草稿器每次验证周期只提交一个草稿序列。一旦在早期位置出现第一个不匹配，后续所有草稿令牌都被丢弃（最长正确前缀规则），导致接受长度受限。简单增加候选分支（如从同一扩散模型采样多个序列）收益有限，因为候选间存在误差同质性，且无法有效覆盖最可能失败的位置。
- **整体含义**：本文旨在设计一种方法，将单个扩散草稿序列转化为结构化的共享前缀候选集，通过置信度引导在关键位置（最可能拒绝边界）定向恢复，从而在不显著增加草拟和验证开销的前提下提高接受长度，实现端到端加速。

## 2. 方法论

### 核心思想
- 采用双扩散草稿范式：第一扩散草稿器（基于 DFlash）生成一个 γ 令牌块并输出每个位置的置信度分数；通过置信度估计拒绝边界的后验分布，选取 Top-K 个最可能拒绝的前缀长度；第二可变前缀扩散草稿器（VP-Drafter）在这些选定前缀处重新锚定，在一次批处理前向中生成替代续接；原始草稿与所有第二草稿候选形成共享前缀结构，通过级联注意力联合验证。

### 关键技术细节
1. **置信度估计与拒绝边界后验**  
   - 每个位置 k 的置信度 \(c_k = \max_v p_k(v)\)，可近似作为该位置令牌被接受的概率。
   - 拒绝边界后验：\(r(i) = \left(\prod_{k=1}^i c_k\right) \cdot (1 - c_{i+1})\)，表示恰好有 i 个令牌被接受的概率。
2. **Top-K 分支选择**  
   - 从 \(i \in \{0, \dots, \gamma-2\}\) 中选取 r(i) 最大的 K 个前缀长度。
   - 每个选定的前缀长度对应一个第二草稿输入：保留锚点及前 i 个 DFlash 生成令牌，将其余位置重新掩码。
3. **可变前缀扩散草稿器（VP-Drafter）训练**  
   - 架构与 DFlash 相同（轻量 Qwen 结构 + KV 特征注入）。
   - 训练时前缀长度 l 从截断几何先验采样（偏向短前缀），前 l+1 个位置填充真实令牌，其余掩码。
   - 损失函数：指数衰减交叉熵，\(w_k = \exp(-(k-l-1)/\tau)\)，强调靠近锚点的位置的重要性。
4. **联合验证**  
   - 所有候选（原始 + K 个第二草稿）共享前缀，通过级联注意力一次验证，选择最长的正确前缀，其后的预测作为下一轮锚点。

## 6. 主要结论与发现

- D^2SD 在数学和代码任务上提升最大（如 GSM8K 上加速比从 5.51× 提升至 6.54×），对话任务提升较小。
- 相比 DFlash，平均接受长度从 5.31 提升至 7.05，加速比从 4.16× 提升至 4.98×（Qwen3-8B, 贪婪解码）。
- 即使 EAGLE-3 在某些数据集上达到略高的接受长度，D^2SD 因并行草稿的低延迟优势仍获得更高加速比。
- 置信度引导的共享前缀恢复优于均匀重采样或单纯扩大块大小（γ 从 16 增至 32 反而降速）。
- 增加第三级联草稿使接受长度提升约 8%，但加速比下降约 4%，表明两级联是合理选择。

## 7. 优点

- **创新性**：首次将扩散草稿与置信度引导的树结构结合，提出“双扩散草稿”范式，针对现有方法的两个失效模式（单序列局限、同质重采样）进行定向优化。
- **高效性**：保持并行草稿的低延迟优势，通过精准重新锚定大幅提高接受长度，级联注意力共享 KV 前缀，验证成本可控。
- **完备消融**：通过三个详细消融实验分别验证了分支策略、第二草稿器设计和级联深度的必要性，说服力强。
- **实用价值**：在主流模型（Qwen3-8B, GPT-OSS-20B）和多种任务上取得一致改进，代码开源。

## 8. 不足与局限

- **算力报告缺失**：未提供训练 GPU 小时数等资源信息，难以评估方法的整体资源效率。
- **模型泛化性有限**：仅测试两个目标模型，且均为 8B/20B 规模；更小或更大模型（如 70B）上的表现未验证。
