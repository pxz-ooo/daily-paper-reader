## 摘要
大语言模型中的自回归解码本质上是内存受限的：每个生成步骤都需要从内存（例如，GPU服务器上的高带宽内存HBM）加载模型权重和中间结果，使得吞吐量受限于内存带宽而非计算能力。推测解码通过并行验证多个草稿令牌来解决这一问题，有效分摊了每次目标模型调用的成本。然而，现有的推测解码方法都假设HBM足够大，能够同时容纳目标模型和辅助草稿模型——这一假设在内存受限设备（例如DRAM有限的边缘平台）上不再成立。我们分析了这种内存受限情况下的推理瓶颈，并提出了CATS，一种自推测解码框架，该框架根据内存预算和内存受限设备上的参数卸载模式进行级联验证和校正。这种设计最大化了令牌接受率和端到端加速，同时将设备上的峰值内存占用保持在仅目标模型本身的大小。我们在真实边缘设备上的五个基准测试中评估了CATS在不同模型上的表现。CATS可实现最高5.08倍的实际加速，且生成质量无下降，在边缘内存约束下比现有最佳方法高出最多1.45倍。

## Abstract
Auto-regressive decoding in Large Language Models (LLMs) is inherently memory-bound: every generation step requires loading the model weights and intermediate results from memory (e.g., High-Bandwidth Memory (HBM) for GPU servers), making throughput bottlenecked by memory bandwidth rather than compute. Speculative decoding addresses this by enabling parallel verification of multiple draft tokens, effectively amortizing the cost of each target-model call. However, existing speculative decoding methods are designed under the assumption that HBM is sufficiently large to hold both the target model and an auxiliary draft model simultaneously -- an assumption that breaks down on memory-constrained devices such as edge platforms with limited DRAM. We analyze the inference bottleneck in this memory-limited regime and propose CATS, a self-speculative decoding framework that conducts cascaded verification and correction based on the memory budget and parameter offloading patterns on memory-limited devices. This design maximizes token acceptance rate and end-to-end speedup while keeping the peak memory footprint on the device equal to that of the target model alone. We evaluate CATS on different models across five benchmarks on real edge devices. CATS can achieve a wall-clock speedup of up to 5.08x with no degradation in generation quality, outperforming the SOTA method by up to 1.45x under edge memory constraints.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：大语言模型（LLM）自回归解码本质上是内存受限的。在服务器 GPU 上，HBM 带宽是瓶颈；在内存有限的边缘设备（如 Jetson AGX Orin，DRAM ≤16 GB）上，模型权重必须从闪存逐块加载到 DRAM，导致**闪存↔DRAM 的数据移动**成为主导延迟因素，其带宽仅为 ~2 GB/s。
- **现有方法的局限**：所有现有推测解码方法（包括使用辅助草稿模型或自推测方法）都假设模型权重可常驻 DRAM。这在边缘设备上不成立：引入额外草稿模型会进一步增加内存占用和传输流量，反而恶化瓶颈。而自推测方法虽无需额外模型，但草稿子网能力有限，且仍需加载额外参数（如适配器）。
- **研究意义**：提出一种专为内存受限环境设计的自推测解码框架 **CATS**，在不增加峰值内存占用（仅目标模型本身）的前提下，通过级联验证大幅提升令牌接受率和端到端加速，使 LLM 能够高效部署于边缘设备。

## 2. 方法论
### 核心思想
- **级联三阶段推理**：将模型分层，根据 DRAM 预算动态划分边界，每个解码周期依次执行：
    1. **草稿阶段（Draft Stage）**：使用浅层子网络（层 1~\(L_{\text{DM}}\)）自回归生成 \(\bar{\gamma}\) 个草稿令牌。草稿模型包括一个轻量适配器（与 Kangaroo 类似）。
    2. **浅层验证阶段（Shallow Verification Stage）**：将草稿令牌并行通过中间层（层 \(L_{\text{DM}}+1\)~\(L_{\text{SV}}\)），由浅层验证器（SV 适配器）产生验证令牌。若验证令牌与草稿令牌匹配则保留；若不匹配，则产生**校正候选**，并重新前传这些校正令牌（复用已加载的 DM 和 SV 层，零额外闪存传输）。
    3. **目标验证阶段（Target Verification Stage）**：流式加载剩余层（层 \(L_{\text{SV}}+1\)~\(L_{\text{final}}\)），将主草稿序列和所有候选分支组装成**验证树**，通过树形掩码注意力一次批量前传，选择长度最长的接受前缀，并输出目标模型的下一个令牌。
- **自适应边界选择**：\(L_{\text{DM}}\) 和 \(L_{\text{SV}}\) 根据 DRAM 容量、计算吞吐量和闪存带宽平衡确定，以**完全消耗可用内存预算**，不改变模型架构。
- **训练损失**：提出 **Reduced KL Loss**，仅对目标分布 top‑\(K\) 令牌进行蒸馏，避免将梯度分散到低概率令牌上，增强小容量适配器的草稿和验证能力。

### 关键技术细节
- 草稿和验证适配器结构：2 个归一化层 + 1 个多头注意力层 + 共享 LM 头（同 Kangaroo）。
- 树形验证：利用树形注意力掩码实现并行批量前传，显著提升每周期接受的令牌数。
- **与 EAGLE 正交**：CATS 可融合 EAGLE 的高概率候选分支，形成 CATS w/ EAGLE，进一步提升接受率。

### 算法流程（文字说明）
1. 预填充阶段：完整模型计算提示词，得到 KV 缓存和初始令牌。
2. 循环直至生成足够令牌：
   - **草稿**：对当前序列重复 \(\bar{\gamma}\) 次：通过层 1~\(L_{\text{DM}}\) 得到隐藏状态，经 DM 适配器解码下一个令牌，自回归生成候选序列。
   - **浅层验证**：将 \(\bar{\gamma}\) 个候选令牌并行通过层 \(L_{\text{DM}}+1\)~\(L_{\text{SV}}\)，SV 适配器生成验证令牌。记录所有不匹配位置作为校正候选；若存在校正，重新前传校正令牌（复用已加载层），构建验证树。
   - **目标验证**：流式加载剩余层，将验证树通过树形掩码注意力批量前传，计算目标分布，按接受准则（贪心或典型接受）遍历树，找到最长接受前缀；接受前缀末尾加上目标模型的下一个预测令牌作为额外令牌。
   - 提交输出，更新 KV 缓存，回滚未接受部分。

## 6. 主要结论与发现
- **性能提升**：
  - Vicuna‑7B 贪心模式下，CATS 平均接受 3.06 令牌，加速 3.18×；相比 Kangaroo（2.27 令牌，1.92×）提升 1.66×。
  - LLaMA2‑7B 贪心模式下，CATS 接受 4.65 令牌，加速 4.65×；CATS w/ EAGLE 接受 5.08 令牌，加速 5.08×，优于所有基线。
  - 松弛解码下增益更大（Vicuna‑7B: 3.79 令牌，加速 3.94×）。
- **内存效率**：CATS 的 BPT（每令牌字节数）显著低于加深草稿模型的 Kangaroo（5.84 vs 11.10 GB/tok），验证了级联验证的优势。
- **质量无损**：在 MT‑bench 上，CATS 处于帕累托前沿（高速度+高质量），生成质量与基线相当。
- **模型无关性**：在 LLaMA2 系列上同样表现优异，证明方法通用。

## 7. 优点
1. **模型无关且无架构修改**：适用于任意 Transformer‑based LLM，无需修改模型内部结构。
2. **自适应内存预算**：通过调整 \(L_{\text{SV}}\) 充分利用可用 DRAM，无需修改其他部分，适应不同边缘设备。
3. **无额外在线模型**：自推测，避免了辅助模型带来的内存和传输开销；峰值内存等同于仅目标模型。
4. **树形验证高效**：通过校正分支构建验证树，仅需一次批前传即可处理多条候选路径，显著提高接受率。
5. **训练高效**：Reduced KL Loss 仅关注高概率令牌，减少了训练容量需求，仅需 2 块 GPU 13 小时即可完成适配器训练。
6. **可扩展性**：可与 EAGLE 等树形方法正交结合，进一步提升性能。

## 8. 不足与局限
1. **模型规模限制**：实验仅涵盖 7B‑13B，更大模型（如 30B/70B）的表现未验证。
2. **预训练依赖**：需要针对每个目标模型训练草稿和验证适配器，存在预训练成本。
3. **架构限制**：假设 Transformer 分层表示，不直接适用于状态空间模型（如 Mamba）或其他非层对齐架构。
4. **评估平台单一**：边缘设备仅使用 Jetson AGX Orin，未在更广泛平台（如 Xavier、树莓派、手机 DSP）上验证。
5. **未考虑大批处理场景**：论文声明专注于小批量的延迟导向推理，当 KV 缓存成为瓶颈时，可类似处理但未深入实验。
6. **比较基线有限**：未包含最新自推测方法（如 Draft & Verify、LayerSkip、KnapSpec）在同等边缘条件下的对比（但已涵盖主流方法 Kangaroo、Medusa、EAGLE 等）。
